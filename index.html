B1

B2

Trung cấp cao

Ý dài hơn, có quan điểm, giải thích, ví dụ và nhiều từ nối hơn.

C1

Nâng cao

Cách thêm audio riêng của bạn

Đây là phần quan trọng nhất của hướng mình chọn: bạn tự quản lý file nghe để tránh rủi ro về bản quyền.

Cấu trúc thư mục gợi ý

/trist2-8.github.io ├─ index.html ├─ audio/ │ ├─ a1-breakfast-at-home.mp3 │ ├─ a2-at-the-bus-stop.mp3 │ ├─ b1-first-job-interview.mp3 │ ├─ b2-teamwork-meeting.mp3 │ ├─ c1-future-of-education.mp3 │ └─ c1-creative-thinking.mp3

Bạn cần làm gì

  1. Giữ file index.html này làm trang chủ của repo GitHub Pages.
  2. Tạo thư mục audio trong repo.
  3. Đặt MP3 theo đúng tên file mà website đang dùng.
  4. Nếu muốn đổi tên file, chỉ cần sửa đường dẫn trong mảng dữ liệu JavaScript ở cuối trang.
  5. Sau khi commit và push, GitHub Pages sẽ tự cập nhật.

Lộ trình tiếp theo

Sau khi bạn có 5-10 file audio đầu tiên, website này đã đủ nền để phát triển thành một nền tảng nghe hoàn chỉnh.

1

Thay nội dung mẫu bằng nội dung thật

Đổi tiêu đề, transcript, từ vựng và quiz trong dữ liệu JavaScript thành bài học thật của bạn.

2

Bổ sung nhiều bài cho từng cấp độ

Mỗi level có 10-20 bài là bạn đã có một thư viện nghe rất ổn để bắt đầu thu hút người học.

3

Tách thành nhiều trang riêng

Sau này có thể làm trang danh sách chuyên sâu, trang bài học riêng, tìm kiếm và lưu tiến độ học.
